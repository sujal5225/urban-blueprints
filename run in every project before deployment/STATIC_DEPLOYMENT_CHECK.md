# Static Site Pre-Deployment Check

## ⚠️ ALL DEVELOPERS: Read Before Final Push

**Auto-deploy is configured.** When you push to `main`, the site automatically builds and deploys.

### You do NOT need to:
- Run `npm run build` locally
- Upload the `/out` folder manually
- Touch any hosting configuration

### You MUST do:
- Run the **Static Export Compatibility Check** (below) before your final push

---

## Pre-Deployment Prompt

Copy and run this prompt in Cursor/AI assistant before pushing:

```
You are helping prepare this Next.js project for static export and automatic deployment to static hosting.

Follow these rules exactly:

1. The project must use Next.js STATIC EXPORT.
   - Configure next.config.js:
        const nextConfig = {
          output: 'export',
          images: { unoptimized: true },
        };
        module.exports = nextConfig;

2. Do NOT use any server-side features.
   - No API routes
   - No getServerSideProps
   - No server actions ('use server')
   - No middleware.ts
   - No filesystem access
   - No Node.js runtime dependencies

3. All pages must work as STATIC pages.
   - Use generateStaticParams() for dynamic routes (App Router)
   - Use getStaticProps / getStaticPaths (Pages Router)
   - Avoid Edge/Node runtime requirements
   - All routing must function without SSR

4. Images must NOT use Next.js image optimization.
   - Use <img> tags or next/image with unoptimized=true
   - No remote loaders that require SSR

5. Prepare package.json for static deployment:
   - Scripts must be:
       "dev": "next dev",
       "build": "next build",
       "preview": "npx serve@latest out",
       "lint": "next lint"
   - Do NOT include "start": "next start" (incompatible with static export)
   - Build command: npm run build
   - Output directory: out

6. The final static build output MUST go into the /out folder.
   - The /out folder is auto-generated by `next build` when output: 'export' is set
   - Do NOT modify anything inside /out manually

7. The repository must be compatible with static hosting:
   - No custom Node server
   - No Express, no APIs, no middleware that depends on SSR
   - Everything MUST work as a client-side or statically generated site

8. When modifying code, always ensure:
   - It can run on any static hosting platform
   - It does not break static export
   - It does not rely on backend features
   - All navigation links point to existing pages

9. Assume DNS and hosting configuration is handled separately.
   - Just ensure the build process is valid
   - Do not add hosting instructions inside the code

Your job:
- Audit the project for static export compatibility
- Convert or update any page that breaks static export
- Suggest required refactoring
- Warn if a feature is not compatible with static export
- Ensure the entire app builds successfully using `npm run build`
- Verify the /out folder is generated with all static HTML files
- Ensure the app is fully deployable on any static hosting platform

Always output clean, production-ready code that works with static hosting.
```

---

## Workflow

1. Make your changes
2. Run the prompt above → Fix any issues found
3. Push to deploy:
   ```bash
   git add .
   git commit -m "your message"
   git push origin main
   ```
4. ✅ Auto-deploy handles the rest

## Common Issues

| Issue | Solution |
|-------|----------|
| `next/image` errors | Add `unoptimized: true` to next.config.js |
| API routes | Remove or convert to client-side fetch |
| `getServerSideProps` | Convert to `getStaticProps` |
| Dynamic routes 404 | Add `generateStaticParams()` |
| Broken nav links | Ensure all href paths have corresponding pages |
